TF-IDF

TF-IDF stands for term frequency, inverse-document frequency. It is a numerical statistic that is intended to reflect how important a word is to a document in a collection or corpus. It follows the bag-of-words model.

Get info about the bag-of-words-model at this link: https://en.wikipedia.org/wiki/Bag-of-words_model (Right-click the link and select 'Open link in new tab').

Get info about tf-idf at this link: https://en.wikipedia.org/wiki/Tf%E2%80%93idf (Right-click the link and select 'Open link in new tab').

If you select TF-IDF option, the list of unique words (after performing stemming or lemmatization if either option is chosen) are extracted. The TF-IDF value of each word in each document in the corpus is calculated. Each document in the corpus can represent each file if multiple files are uploaded. It can represent each paragraph if one file is uploaded. If the word doesn't exist in the specific file or paragraph the TF-IDF value is calculated as 0.

You can see the results in the Display Data tab.

LDA

LDA stands for latent Dirichlet allocation. It s a generative statistical model that allows sets of observations to be explained by unobserved groups that explain why some parts of the data are similar.

Get info about LDA at this link: https://en.wikipedia.org/wiki/Latent_Dirichlet_allocation (Right-click the link and select 'Open link in new tab').

When you choose the LDA option, you can choose the number of topics you want the list of words to be grouped in. Each Topic becomes a column (Topic1, Topic2, and so on) and the LDA values of each topic in each document in the document corpus is calculated.

Just like TF-IDF calculation, each document can represent each paragraph if 1 file is uploaded or 1 file if multiple files are uploaded. The result table can be seen in the Display Data tab. If you want to see the number of words in each topic, you can choose the Topic Modelling option in the Data Modelling - Unstructured tab.
